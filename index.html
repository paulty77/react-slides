Composants

Chaque élement de la page est représenté par un composant qui est un fichier javascript.

On va faire de la composition et imbriquer les composants pour constituer la page

Etat et propriétés

Chaque composant a un état qui va agir sur son rendu.

Chaque composant peut aussi recevoir des propriétés qui peuvent agir sur son état.

On aura donc une notion d'état global de l'application réprésentant tous les états des composants.

Isolation et testabilité

Comme chaque composant est un isolé, on peut tester plus facilement un code de test par composant.

DOM virtuel

On ne manipule pas directement le DOM mais on va utiliser un DOM virtuel qui se chargera d'intervenir sur le DOM uniquement si il y a besoin.

Composants présentationnels et Conteneurs

2 catégories de composants :

1 = Stateful / Conteneur / Applicatif

Spécifique à l’application, « connecté » à son état global, doté de gestionnaires d’événements spécifiques.

2 = Stateless / Composant « bête »

Générique et réutilisable, ne voit que les props qu’on lui passe, aucune logique applicative interne.

Unidirectionnel

Flux dans un seul sens donc plus facile à débugger

Comment fonctionne React ?

JSX

On va écrire du langage ressemblant à du HTML dans le javascript.

Syntaxe JSX:


										render () {
										  return <input type='email' name={name} />
										}

Syntaxe javascript:


										render () {
										  return React.createElement('input', { type: 'email', name })
										}

Les éléments de Redux

Store : objet unique qui contient un objet "état global de l'application" et des reducers

Reducers : fonctions pures qui reçoit un état en paramètre et qui renvoie un nouvel état en paramètre

Actions : fonctions qui renvoie un objet avec un type et d'autres valeurs qui vont servir aux reducers

Dispatch : fonction qui va permettre à l'UI d'appeler une action qui va déclencher l'appel à un réducer
